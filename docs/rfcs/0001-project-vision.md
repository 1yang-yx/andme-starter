> 状态：已通过

---

## 一句话定位

andme-starter 是一个开箱即用的微信小程序模板，帮助开发者跳过重复搭建，直接写业务。

## 目标用户

**主要用户：独立开发者和初级前端开发者**

- 一个人做小程序，没有团队，想快速上线产品
- 刚学 Vue，想做第一个小程序，需要"能跑的样板代码"
- 不想从零搭工程，不想每次都重写登录、列表、表单

## 解决的痛点（按优先级）

1. **每次新项目都要从零搭工程** — request 封装、登录、页面骨架、manifest 配置，全是重复劳动
2. **找不到一个"能跑的完整项目"参考** — 网上教程都是片段代码，没有从登录到列表到详情的完整闭环
3. **列表/表单/详情每次都重写** — 下拉刷新、上拉加载、空状态、loading、错误重试，每个项目写一遍

## 核心卖点（vs 竞品）

| 维度 | unibest | andme-starter |
|------|---------|---------------|
| 定位 | 工程化脚手架 | 业务样板模板 |
| 开箱体验 | 空项目 + 配置好的工具链 | 5 个完整业务页 + 能跑的 Demo |
| 后端支持 | 不涉及 | Mock / Supabase / 自定义适配器 |
| 上线指导 | 无 | 微信过审 checklist |
| 真实案例 | 无 | 食谱小程序（已上线） |

**一句话差异化**：unibest 给你"搭好的脚手架"，andme-starter 给你"写好的样板业务"。

- [ ] .env 环境变量管理

## 明确不做（v1.0）

- ❌ 多端适配（v2.0 路线图，当前只做微信小程序）
- ❌ UI 组件库（用现成的，不造轮子）
- ❌ 代码生成器（只提供样板，用户复制修改）
- ❌ 后端代码（只做前端模板 + 适配器接口）
- ❌ 企业级功能（权限 / 角色 / 审批流）
- ❌ 国际化
- ❌ 状态管理强约束（不强制 Pinia/Vuex）

## 成功标准

- GitHub 200 Star
- 30 个真实用户，有社区反馈
- 不设 deadline，质量优先

## 风险与应对

### 风险 1：代码质量不够，用户不敢用
**应对**：
- 每个模块用食谱 Demo 真实验证后再提交
- 核心 hooks 写单元测试（Vitest）
- ESLint + Prettier 强制代码风格统一

### 风险 2：和 unibest 差距太大，用户不选我
**应对**：
- 不在"工程化全面性"上和 unibest 竞争
- 差异化打"业务样板 + 过审指南 + 真实上线案例"
- README 明确写出和 unibest 的定位区别

## 技术栈

- **框架**：uni-app + Vue 3 + TypeScript
- **构建**：Vite
- **包管理**：pnpm
- **代码规范**：ESLint + Prettier
- **CSS**：SCSS（不用 UnoCSS / Tailwind，降低门槛）
- **状态管理**：Vue ref + provide/inject（不强制 Pinia）
- **HTTP**：uni.request 封装（不引入 axios）
- **测试**：Vitest（核心 hooks）
